# 프로젝트 규칙 (GEMINI.md)

이 문서는 프로젝트 개발 시 AI 에이전트(Google Antigravity)가 준수해야 하는 규칙을 정리한 문서입니다.

## 기본 규칙 (Rules)

1. 지금부터 한글로 해줘
2. 코드를 생성하거나 수정할때 항상 주석을 한글로 추가해 
3. .env 파일은 보안상의 이유로 절대 읽거나 수정하지 마. 
   대신 설정 값이 필요하면 .env-sample 만들고 .env-example 파일만 수정해.
4. 요청을 실행할 때 항상 계획을 먼저 알려주고 , 승인하면 요청을 시행하자.
5. 항상 마직막 작업 내용을 루트디렉토리에 /doc/마지막작업-오늘날짜.md 파일로 써줘 , 기존 내용 유지하고 추가하는 거야  
6. git 푸시는 내가 요청할때 하자 알았지 ? 
7. git 커밋은 니가 자동으로 해 
8. git 코맨트는 한글로 하자 
9. 파일삭제는 절대로 물어보고 해야해 알았지 ?
10. 항상 계획을 파일로 만들어 알려 주고 , 승인 하면 작업 해 

## 세부 규칙

### 1. 명령어 자동 실행 (SafeToAutoRun)
- **자동 실행 허용**: 단순 파일 조회(`ls`, `find`, `grep`), 빌드 테스트(`xcodebuild`), 네트워크 상태 확인 등 파일 내용을 직접적으로 수정하거나 삭제하지 않는 명령어는 승인 없이 즉시 실행한다.
- **승인 필수**: 파일 삭제(`rm`), Git 커밋/푸시(`git commit/push`), 코드 수정 작업은 반드시 사전에 계획을 알리고 승인을 받은 후 실행한다.

### 2. 명령어 포맷
- 터미널 명령어 실행 시, 사용자가 엔터를 여러 번 치지 않도록 가급적 줄바꿈(`\`) 없이 한 줄로 합쳐서 실행하거나 깔끔한 포맷을 유지한다.

## ⚙️ Google Antigravity 에이전트 룰 적용 및 수정 방법

이 규칙들을 Google Antigravity 에이전트가 항상 인지하도록 적용하는 방법입니다.

### 1. UI (Customizations)를 통한 적용 및 수정
- Antigravity 에이전트 채팅창 우측 상단의 **더보기 메뉴(•••)**를 클릭합니다.
- **Customizations** ⚙️ 메뉴를 선택합니다.
- **Rules** 섹션에서 새로운 룰을 직접 추가하거나, 기존 룰을 편집할 수 있습니다.

### 2. 로컬 파일 시스템을 통한 적용 및 수정 (추천)
에이전트가 기동될 때 자동으로 룰을 읽어오도록 파일로 지정할 수 있습니다.

- **전역(Global) 규칙 설정**:
  - 파일 경로: `~/.gemini/GEMINI.md` (사용자 홈 디렉토리 아래 `.gemini` 폴더 내부)
  - 이 경로에 본 파일의 내용을 복사해 두면, PC에서 기동되는 **모든 프로젝트**의 Antigravity 에이전트에 동일한 규칙이 자동 적용됩니다.
- **프로젝트(Workspace) 규칙 설정**:
  - 파일 경로: `[프로젝트 루트]/.agents/rules/gemini_rules.md` (또는 하위 호환을 위해 `.agent/rules/` 폴더도 지원)
  - 이 경로에 마크다운 파일로 작성해 두면, **해당 프로젝트** 내에서 작업할 때만 Antigravity 에이전트가 룰을 적용합니다.

> [!NOTE]
> 규칙을 수정한 후에는 에이전트가 변경 사항을 인지할 수 있도록 에이전트를 새로고침(Refresh)하거나 대화를 새로 시작하는 것이 좋습니다.
