# 강의용 개발 내용 예제

## 1. 프로젝트 개요
### 목표
- `./강의자료` 폴더 내 강의 콘텐츠를 서비스로 전환
- 수강생이 강의자료를 검색, 열람, 다운로드할 수 있는 웹 인터페이스 제공
- 서버리스 백엔드 + 정적 프론트엔드 구조로 빠른 배포 및 유지보수

### 주요 요구사항
- 강의자료 목록 조회
- 강의자료 검색 기능
- 자료 상세보기 및 다운로드
- 관리자 자료 등록/수정/삭제
- 배포 환경에서 안전하게 동작

## 2. 기술 스택
- 프론트엔드: Vite + Vue 3 + TypeScript
- 백엔드: Node.js ESM + Express (Vercel Serverless Functions)
- 데이터베이스: Turso Cloud DB + `@libsql/client/web`
- 배포: Vercel
- 환경설정: `./doc/개발환경.md` 참조

## 3. 강의 예제 구성
### 3-1. 프론트엔드 구성
- `src/pages/Home.vue`
  - 강의자료 목록 조회, 검색 입력, 필터
- `src/components/LectureCard.vue`
  - 강의자료 카드 UI
- `src/pages/Detail.vue`
  - 자료 상세보기 및 다운로드 버튼
- `src/pages/Admin.vue`
  - 자료 등록/수정/삭제 화면

### 3-2. 백엔드 구성
- `api/db.ts`
  - Turso DB 초기화, 공통 DB 연결 함수
- `api/lectures.ts`
  - 강의자료 CRUD API 엔드포인트
- `api/middleware/initDb.ts`
  - 서버리스 콜드 스타트 시 DB 초기화를 보장하는 미들웨어

## 4. API 설계 예제
### 4-1. 강의자료 목록 조회
- HTTP 메서드: `GET`
- 경로: `/api/lectures`
- 설명: 등록된 강의자료 목록 반환
- 쿼리 파라미터: `query`(선택), `category`(선택)

### 4-2. 강의자료 상세 조회
- HTTP 메서드: `GET`
- 경로: `/api/lectures/:id`
- 설명: 특정 강의자료 정보 반환

### 4-3. 강의자료 등록
- HTTP 메서드: `POST`
- 경로: `/api/lectures`
- 설명: 새 강의자료 등록
- 요청 본문 예시:
  ```json
  {
    "title": "Vue 3 기초 강의",
    "category": "프론트엔드",
    "description": "Composition API를 활용한 웹 개발 입문",
    "fileUrl": "https://example.com/lecture1.pdf"
  }
  ```

### 4-4. 강의자료 삭제
- HTTP 메서드: `DELETE`
- 경로: `/api/lectures/:id`
- 설명: 등록된 강의자료 삭제

## 5. 예제 아키텍처 노트
### 5-1. 데이터 모델 예시
- `lectures` 테이블 필드
  - `id` (문자열 또는 숫자)
  - `title`
  - `category`
  - `description`
  - `fileUrl`
  - `createdAt`

### 5-2. 서버리스 환경 고려
- Vercel 함수는 cold start가 있을 수 있음
- DB 연결은 재사용하고, 초기에 한 번만 초기화
- `@libsql/client/web` 사용 시 서버리스 환경에서 HTTP 기반 Turso 연결이 안정적

### 5-3. 강의용 코드 예제 포인트
- `GET /api/lectures?query=` 형태의 검색용 API 구현
- 프론트에서 `fetch()` 또는 `axios`로 API 호출
- `LectureCard` 컴포넌트에 자료 요약과 다운로드 버튼 배치

## 6. 강의용 실습 흐름
1. 환경 설정 확인 (`./doc/개발환경.md` 참고)
2. 데이터베이스 모델 설계 및 초기화 함수 작성
3. API 엔드포인트 구현
4. 프론트엔드에서 목록 조회 및 검색 UI 구현
5. 상세 페이지와 다운로드 기능 연결
6. 관리자 등록/수정/삭제 화면 추가
7. Vercel 배포 및 동작 확인
