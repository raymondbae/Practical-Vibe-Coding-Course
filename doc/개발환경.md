
## 1. 기술 스택 & 서버리스 아키텍처 설계 (🚨 극복 노하우 100% 반영)
- **프론트엔드**: Vite + Vue 3 (Composition API) + TypeScript + Vanilla CSS (최신 디자인 요소 반영)
- **백엔드 (API)**: Node.js ESM (Express) + Vercel Serverless Functions
  - 모든 API 내부 상대 경로 임포트 시 **`.js` 확장자 명시** (예: `import { db } from "./db.js"`)
- **데이터베이스**: Turso Cloud DB
  - 네이티브 C++ 바인딩을 차단하고 500 에러를 막기 위해 **`@libsql/client/web`** 초경량 드라이버 탑재.
  - ESM 환경에 맞게 `require`를 완전히 소멸시키고 **`await import(...)`** 및 Proxy 지연 로딩 패턴 결합.
  - 콜드 스타트 시 데이터 누락 방지를 위한 **데이터베이스 동기화 보장 미들웨어(Sync Lock Middleware)** 도입.
- **배포**: Vercel
- **환경변수**: .env-sample 예제 사용 
- **Git**:.env 파일 사용하지 않음. (예제만 작성)
## 2. 세부 아키텍처 구현 방안

### ① Vercel Serverless Function 500 에러 차단
- Turso DB 연동 시 `@libsql/client` 대신 **`@libsql/client/web`** 패키지를 설치 및 임포트하여 순수 HTTP REST 프로토콜로 분산 SQLite 데이터베이스에 연결합니다.

### ② ESM 표준 확장자 생략 버그 해결
- `api/` 내부의 모든 파일 간 임포트 시, 빌드된 파일 기준으로 올바른 런타임 해석이 가능하도록 다음과 같이 `.js` 확장자를 반드시 포함합니다.
  ```typescript
  import { initDatabase } from './db.js';
  import { analyzeAccount } from './services/analyzer.js';
  ```

### ③ 데이터베이스 동기화 보장 미들웨어 설계
- API 서버 구동 및 콜드 스타트 시, `initDatabase()`가 완전히 마무리되기 전에 사용자의 분석 쿼리가 호출되는 것을 차단하는 **동기화 락(Lock) 미들웨어**를 Express 라우터단에 구성합니다.
  ```typescript
  let isDbInitialized = false;
  let dbInitializationPromise: Promise<void> | null = null;

  export async function ensureDbInitialized(req: any, res: any, next: any) {
    if (!isDbInitialized) {
      if (!dbInitializationPromise) {
        dbInitializationPromise = initDatabase().then(() => {
          isDbInitialized = true;
        });
      }
      await dbInitializationPromise;
    }
    next();
  }
  ```
