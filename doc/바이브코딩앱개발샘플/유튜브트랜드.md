유튜브 Data API를 활용해서 한국 인기 급상승 동영상을 가져오는 웹 앱을 만들 거야. 구글 API 할당량을 아끼기 위해, 1시간에 한 번만 유튜브 API를 호출해서  DB에 저장하고, 유저가 접속하면 유튜브가 아닌 내 DB에서 데이터를 읽어오도록 대시보드를 짜줘.


## 2. 유튜브 트렌드 앱 빌드할 때의 구체적 설계 아키텍처

```
[유튜브 API / 혹은 크롤러] 
       ↓ (1시간에 딱 1번 실행 - GitHub Actions활용)
[Vercel Serverless Function] 
       ↓ (데이터 가공 후 저장)
[Turso DB (Edge SQLite)] 
       ↓ (유저가 사이트 접속 시)
[Vercel Frontend -> 유저에게 0.1초 만에 화면 표시!

## 🛠️ 바이브 코딩용 'GitHub Actions 크론 세팅' 코드

나중에 웹 앱 다 만드시고 Vercel에 배포한 뒤, Cursor나 AI 도구에게 아래 코드를 참고해서 세팅해 달라고 하시면 됩니다.

내 깃허브 저장소 폴더에 `.github/workflows/youtube-cron.yml` 파일 하나만 만들고 아래처럼 적어두면 끝입니다.

```yaml
name: YouTube Trend Fetch Cron

on:
  schedule:
    # 매 1시간마다 실행 (UTC 기준이라 한국 시간 고려 안 해도 주기적으로 돔)
    - cron: '0 * * * *'

jobs:
  trigger-vercel-api:
    runs-on: ubuntu-latest
    steps:
      - name: Call Vercel Cron API
        run: |
          curl -X POST "https://내-버셀-앱-주소.vercel.app/api/cron" \
          -H "Authorization: Bearer ${{ secrets.CRON_SECRET }}"

```

> **보안 팁:** 그냥 주소만 노출해 두면 해커나 다른 사람들이 내 Vercel 주소를 마구 찔러서 데이터베이스를 망가뜨릴 수 있으니, 비밀번호(`CRON_SECRET`) 같은 보안 키를 깃허브 시크릿에 등록해 두고 연동하는 구조로 짜달라고 AI에게 지시하시면 완벽합니다.
