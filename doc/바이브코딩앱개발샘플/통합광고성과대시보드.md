# 📊 온라인 광고 부서용 실무 앱 기획: Vibe-Ad Board
> **통합 광고 성과 차트 대시보드 (Vibe-Ad Board)**

본 기획 사양서는 온라인 광고 사업부 마케터들이 매일 여러 매체(Meta, Google 등)에서 쏟아지는 광고 성과 리포트를 수작업으로 취합하고 비교하는 비효율을 해결하기 위해 기획되었습니다. 

개발 실습 진행 시 복잡한 백엔드나 데이터베이스 연동 없이, 마케터가 추출한 실무 CSV 파일을 업로드하여 브라우저단에서 고급스러운 차트와 핵심 성과 지표(KPI)로 시각화해 주는 정적 대시보드 앱으로 구성합니다.

---

## 🎯 1. 프로젝트 개요 & 비즈니스 가치

* **현업의 문제점**: 마케터는 Meta 광고 관리자, Google 광고 관리자 등 서로 다른 매체에 매번 로그인하여 리포트를 다운로드하고, 엑셀을 켜서 수식(ROAS, CPC, CTR)을 직접 걸어 성과를 취합해야 하므로 리포팅 업무에 지나치게 많은 시간이 소요됩니다.
* **해결 방안**: 정해진 형식의 성과 리포트 CSV 파일을 앱에 **드래그 앤 드롭**하기만 하면, 즉시 데이터가 파싱되어 인터랙티브한 차트와 핵심 성과 카드로 실시간 렌더링되는 웹 대시보드를 제공합니다.
* **핵심 이점**: 
  - 백엔드/DB 의존성이 전혀 없는 안전하고 빠른 클라이언트 측(Client-Side) 파싱 및 분석.
  - 마케팅 의사결정을 돕는 아름다운 비주얼과 극도의 단순성.
  - 모바일과 PC 모두에 완벽히 최적화된 반응형 UI로 임원진 보고나 외부 이동 시에도 성과 상시 확인 가능.

## 🖥️ 2. 핵심 UI/UX 및 기능 요구사항

### 2-1. 메인 대시보드 화면 구성
1. **헤더 및 파일 드롭존 (Drop Zone)**
   - 화면 상단에 위치한 미니멀하고 직관적인 드래그 앤 드롭 영역.
   - CSV 파일을 끌어다 놓거나 클릭하여 선택하면 즉시 분석이 시작되는 역동적인 모션 효과 부여.
   - 다운로드받아 바로 테스트해 볼 수 있는 **[실습용 광고성과 테스트 CSV 파일](file:///Users/bjhhmc/github.com/Practical-Vibe-Coding-Course/doc/바이브코딩앱개발샘플/광고성과_테스트샘플.csv)** 다운로드 링크 배치.
   - 모바일 접속을 돕는 **QR 코드 레이어 팝업** 버튼 내장 (QR 코드를 통해 스마트폰에서 대시보드에 즉시 진입).

2. **주요 성과 카드 (Core KPI Summary Cards)**
   - 업로드된 데이터를 기반으로 다음 4가지 핵심 마케팅 지표를 산출하여 카드 형태로 시각화:
     - **총 광고 집행비 (Total Spend)**: 전체 사용 예산 합계.
     - **총 매출액 (Total Revenue)**: 광고를 통한 총 구매 전환 매출액 합계.
     - **평균 ROAS (광고비 대비 매출액)**: `(총 매출액 / 총 광고비) * 100` (%)
     - **평균 CPC (클릭당 비용)**: `총 광고비 / 총 클릭수` (원)

3. **인터랙티브 분석 차트 (Interactive Performance Charts)**
   - **일별 광고비 & 매출액 추이 (듀얼 축 라인/바 차트)**: 기간별 비용 대비 성과가 어떻게 성장하고 있는지 분석.
   - **매체별 점유율 (도넛 차트)**: Meta와 Google 중 어느 매체에 비용을 많이 썼고 매출이 어디서 많이 났는지 시각적 비교.
   - **캠페인별 성과 랭킹 (가로 바 차트)**: 클릭률(CTR)과 ROAS 기준 상위 5개 캠페인을 정렬하여 노출.

4. **성과 원본 데이터 테이블 & 필터링**
   - 업로드된 원본 행 데이터를 표 형태로 하단에 제공.
   - 매체별(Meta, Google 등) 및 날짜 범위 필터를 제공하여 관심 있는 데이터만 신속하게 정렬 및 필터링 가능.

## 💾 3. 파일 업로드 포맷 (CSV 스키마)

업로드할 CSV 파일은 아래와 같은 표준 열(Column) 헤더와 데이터 형식을 따릅니다. 실습에 활용할 가상 데이터는 이미 `광고성과_테스트샘플.csv` 파일로 준비되어 있습니다.

| 열 이름 (Header) | 데이터 타입 | 설명 및 예시 |
| :--- | :--- | :--- |
| `Date` | `YYYY-MM-DD` | 광고 집행 일자 (예: `2026-05-01`) |
| `Channel` | `String` | 광고 매체 플랫폼 (예: `Meta`, `Google`) |
| `Campaign` | `String` | 광고 캠페인명 (예: `[블랙프라이데이] 브랜드 인지도 캠페인`) |
| `Impressions` | `Integer` | 광고가 노출된 횟수 (예: `45000`) |
| `Clicks` | `Integer` | 소비자가 광고를 클릭한 횟수 (예: `980`) |
| `Spend` | `Integer` | 실제 소진된 일일 광고비 (단위: 원, 예: `150000`) |
| `Revenue` | `Integer` | 광고를 통해 일어난 구매 매출 (단위: 원, 예: `600000`) |
| `Conversions` | `Integer` | 최종 전환 건수 (예: `12`) |

## 🛠️ 4. 추천 기술 스택 & 구현 가이드라인

* **프론트엔드 프레임워크**: `Vite + Vue 3` 또는 `순수 HTML/JS` (실무 교육 환경에 맞는 최적 도구 선택).
* **데이터 파싱**: `PapaParse` 라이브러리를 적극 활용하여 브라우저에서 CSV를 신속하고 에러 없이 JSON 객체 어레이로 파싱.
* **Vercel 파일 업로드 아키텍처 최적화 (★)**:
  - Vercel과 같은 서버리스 호스팅 환경에서는 백엔드 임시 폴더에 대용량 파일을 쓰고 저장하는 행위가 제한됩니다.
  - 이를 우려할 필요 없이, **클라이언트 사이드 파싱(Client-Side Parsing)** 방식을 도입해 사용자가 업로드한 CSV 파일을 서버로 전송하지 않고 브라우저 메모리상에서 바로 JS 객체로 전환하여 분석합니다.
  - 이 방식은 **민감한 회사 성과 데이터의 유출을 원천 차단(보안상 매우 우수)**하며, 서버 통신 지연이 전혀 없어 **초고속으로 차트를 렌더링**하고 Vercel 무료 티어에서도 평생 트래픽 제한 없이 완벽하게 구동 가능합니다.
* **차트 시각화**: `Chart.js` (또는 `ApexCharts`)를 활용하여 다크 모드를 지원하는 현대적이고 미려한 인터랙티브 그래픽 구성.
* **디자인 & 테마**: 
  - **다크/글래스모피즘(Glassmorphic) 테마** 적용: 딥 블루/퍼플 계열 그라데이션 바탕과 반투명 백그라운드 카드로 모던하고 하이엔드한 느낌을 극대화.
  - 마우스를 올릴 때 미세하게 떠오르는 호버(Hover) 애니메이션 적용.
* **배포**: Vercel 연동을 통한 즉시 배포 환경 구성.
* **보호 규정 준수**: 전체 개발 과정에서 `doc` 폴더는 영구적으로 보호 및 보존하며, 생성된 테스트 자료 링크를 모바일 테스트가 편리하도록 QR 코드로 웹 화면상에 직접 렌더링되게 구성.

## 📝 5. 핵심 계산식 레퍼런스 (마케팅 수식)

마케팅 대시보드를 구성할 때 JavaScript에서 각 지표를 요약하기 위해 아래 수식을 엄격히 적용해 코딩합니다.

* **CTR (클릭률, %)**
  $$\text{CTR} = \left( \frac{\sum \text{Clicks}}{\sum \text{Impressions}} \right) \times 100$$
* **CPC (클릭당 비용, 원)**
  $$\text{CPC} = \frac{\sum \text{Spend}}{\sum \text{Clicks}}$$
* **ROAS (광고 대비 매출액, %)**
  $$\text{ROAS} = \left( \frac{\sum \text{Revenue}}{\sum \text{Spend}} \right) \times 100$$
* **CVR (전환율, %)**
  $$\text{CVR} = \left( \frac{\sum \text{Conversions}}{\sum \text{Clicks}} \right) \times 100$$
